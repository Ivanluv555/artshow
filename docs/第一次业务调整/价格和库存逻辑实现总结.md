## ✅ 已完成功能

### 1. 库存管理系统

#### 1.1 ProductRepository 扩展
新增了原子性的库存操作方法：

- `deductStock(productId, quantity)` - 扣减库存（使用WHERE条件防止超卖）
- `restoreStock(productId, quantity)` - 恢复库存（取消订单时使用）

```java
@Modifying
@Query("UPDATE Product p SET p.stock = p.stock - :quantity WHERE p.id = :productId AND p.stock >= :quantity")
int deductStock(@Param("productId") Integer productId, @Param("quantity") Integer quantity);
```

#### 1.2 ProductService 新增方法
实现了库存管理的业务逻辑：

- `checkStock(productId, quantity)` - 检查库存是否充足
- `deductStock(productId, quantity)` - 扣减库存
- `restoreStock(productId, quantity)` - 恢复库存

特性：
- 参数验证（null检查、数量必须大于0）
- 使用 `@Transactional` 保证事务一致性
- 原子操作防止超卖

### 2. 完整订单流程

#### 2.1 从购物车创建订单
实现了 `OrderService.createOrderFromCart()` 方法，包含以下步骤：

1. **验证购物车项** - 确保所有购物车项属于当前用户
2. **检查商品库存** - 在创建订单前检查所有商品库存是否充足
3. **生成唯一订单号** - 格式：`ORD + yyyyMMddHHmmss + 4位随机数`
4. **创建订单主表** - 设置订单状态为 `pending`（待支付）
5. **创建地址快照** - 保存收货地址的历史快照
6. **创建订单项** - 保存以下快照信息：
   - `priceAtPurchase` - 购买时的价格快照
   - `productNameSnapshot` - 商品名称快照
   - `productImageSnapshot` - 商品图片快照
7. **扣减库存** - 使用原子操作防止超卖
8. **清空购物车** - 删除已结算的购物车项

整个流程使用 `@Transactional` 保证原子性，任何步骤失败都会回滚。

#### 2.2 订单状态管理
实现了完整的订单状态流转方法：

- `cancelOrder(orderId)` - 取消订单并恢复库存
  - 仅允许 `pending` 或 `paid` 状态的订单取消
  - 自动恢复所有订单项的库存
  
- `payOrder(orderId)` - 支付订单
  - 将订单状态从 `pending` 改为 `paid`
  
- `shipOrder(orderId)` - 发货
  - 将订单状态从 `paid` 改为 `shipped`
  
- `completeOrder(orderId)` - 完成订单
  - 将订单状态从 `shipped` 改为 `completed`

所有方法都包含：
- 权限检查（仅订单所有者可操作）
- 状态验证（确保状态流转合法）
- 事务保证

## 📊 数据快照机制

- `priceAtPurchase` - 购买时价格（商品价格可能变动）

## 🔧 技术要点

### 1. 原子性库存扣减
```java
@Query("UPDATE Product p SET p.stock = p.stock - :quantity 
       WHERE p.id = :productId AND p.stock >= :quantity")
int deductStock(@Param("productId") Integer productId, 
                @Param("quantity") Integer quantity);
```

关键点：
- `WHERE p.stock >= :quantity` 确保只有库存充足时才扣减
- 返回影响行数，0表示库存不足
- 单条SQL执行，避免并发问题

### 2. 订单号生成
```java
private String generateOrderNumber() {
    String timestamp = LocalDateTime.now()
        .format(DateTimeFormatter.ofPattern("yyyyMMddHHmmss"));
    int randomNum = new Random().nextInt(9000) + 1000; // 1000-9999
    return "ORD" + timestamp + randomNum;
}
```

格式示例：`ORD202607061015301234`
- 前缀：ORD
- 时间戳：14位（精确到秒）
- 随机数：4位（防止同一秒内冲突）

### 3. 事务管理
所有涉及多表操作的方法都使用 `@Transactional` 注解：
- `createOrderFromCart` - 订单创建流程
- `cancelOrder` - 取消订单恢复库存
- `deductStock` / `restoreStock` - 库存操作

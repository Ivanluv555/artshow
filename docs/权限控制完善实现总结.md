## ✅ 已完成功能

### 1. Post（帖子）权限控制

#### 1.1 修复的安全漏洞
**原问题：**
- 任何用户可以删除或修改他人的帖子
- 没有检查当前用户是否是帖子作者

**修复方案：**

**修改文件：** `PostService.java`

**新增权限检查：**
```java
// 删除帖子
public void deletePost(Long PostId) {
    // 查询帖子
    Post post = postRepository.findById(PostId)
            .orElseThrow(() -> new BizException(ResultCodes.NOTFOUND, "帖子不存在"));

    // 权限检查：只有帖子作者或管理员可以删除
    Long currentUserId = UserContext.getUserId();
    String currentUserRole = UserContext.getRole();

    if (!post.getUserId().equals(currentUserId) && !"ADMIN".equals(currentUserRole)) {
        throw new BizException(ResultCodes.FORBIDDEN, "无权删除此帖子");
    }

    postRepository.deleteById(PostId);
}

// 修改帖子
public Post updatePost(PostDTO Post) {
    // 查询帖子
    Post post = postRepository.findById(postId)
            .orElseThrow(() -> new BizException(ResultCodes.NOTFOUND, "帖子不存在"));

    // 权限检查：只有帖子作者或管理员可以修改
    Long currentUserId = UserContext.getUserId();
    String currentUserRole = UserContext.getRole();

    if (!post.getUserId().equals(currentUserId) && !"ADMIN".equals(currentUserRole)) {
        throw new BizException(ResultCodes.FORBIDDEN, "无权修改此帖子");
    }

    // 更新帖子内容（不允许修改userId）
    if (Post.getTitle() != null) post.setTitle(Post.getTitle());
    if (Post.getDescription() != null) post.setDescription(Post.getDescription());
    if (Post.getImageUrl() != null) post.setImageUrl(Post.getImageUrl());
    if (Post.getSubcategoryId() != null) post.setSubcategoryId(Post.getSubcategoryId());

    return postRepository.save(post);
}
```

**安全特性：**
- ✅ 只有作者可以修改/删除自己的帖子
- ✅ 管理员可以修改/删除任何帖子
- ✅ 禁止修改帖子的作者（userId字段）
- ✅ 资源不存在时返回明确错误
- ✅ 无权限时返回403 FORBIDDEN

## 🔄 权限检查流程

### 标准权限检查模式

```java
// 1. 获取资源
Resource resource = repository.findById(resourceId)
    .orElseThrow(() -> new BizException(ResultCodes.NOTFOUND, "资源不存在"));

// 2. 获取当前用户信息
Long currentUserId = UserContext.getUserId();
String currentUserRole = UserContext.getRole();

// 3. 权限验证
boolean isOwner = resource.getOwnerId().equals(currentUserId);
boolean isAdmin = "ADMIN".equals(currentUserRole);

if (!isOwner && !isAdmin) {
    throw new BizException(ResultCodes.FORBIDDEN, "无权操作此资源");
}

// 4. 执行操作
// ... 业务逻辑
```

### 权限检查时序

```
客户端请求
    ↓
AuthInterceptor（JWT验证）
    ↓
提取userId和role存入UserContext
    ↓
Controller（路由处理）
    ↓
Service（权限检查）
    ├─ 查询资源
    ├─ 获取当前用户信息
    ├─ 比对所有权
    └─ 验证通过 → 执行操作
       验证失败 → 抛出FORBIDDEN异常
    ↓
返回结果或错误
```

## 🛡️ 安全特性

### 1. 资源所有权验证

**机制：**
- 每次修改/删除操作前先查询资源
- 比对资源的所有者ID与当前用户ID
- 不匹配则拒绝操作

**代码示例：**
```java
if (!resource.getOwnerId().equals(currentUserId) && !"ADMIN".equals(currentUserRole)) {
    throw new BizException(ResultCodes.FORBIDDEN, "无权操作此资源");
}
```

### 2. 管理员特权

**特性：**
- 管理员可以绕过所有权检查
- 用于内容审核、违规处理
- 所有权限检查都包含管理员判断

**代码模式：**
```java
boolean hasPermission = isOwner || "ADMIN".equals(currentUserRole);
```

### 3. 禁止篡改关键字段

**保护的字段：**
- Post: `userId`（作者不可改）
- Product: `sellerId`（卖家不可改）
- Comment: `userId`、`postId`（作者和关联帖子不可改）

**实现方式：**
```java
// 使用选择性更新而非BeanUtils.copyProperties
if (dto.getTitle() != null) entity.setTitle(dto.getTitle());
if (dto.getDescription() != null) entity.setDescription(dto.getDescription());
// 不更新 userId, sellerId 等关键字段
```

### 4. 明确的错误提示

**错误类型：**
- `404 NOTFOUND` - 资源不存在
- `403 FORBIDDEN` - 无权限操作
- `401 UNAUTHORIZED` - 未登录

**示例：**
```java
throw new BizException(ResultCodes.NOTFOUND, "帖子不存在");
throw new BizException(ResultCodes.FORBIDDEN, "无权删除此帖子");
```

## ⚠️ 注意事项

### 1. 创建资源时的所有者设置

**当前实现：**
- 创建帖子/评论时，需要前端传入`userId`
- 存在安全风险：用户可能伪造其他人的ID

**建议改进：**
```java
@Override
public Post addPost(PostDTO post) {
    Post nPost = new Post();
    BeanUtils.copyProperties(post, nPost);
    
    // 强制使用当前登录用户的ID，忽略前端传入的userId
    nPost.setUserId(UserContext.getUserId());
    
    return postRepository.save(nPost);
}
```

### 2. 批量操作的权限控制

**当前状态：**
- `deleteProducts(Iterable<Long> productIdList)` 没有权限检查

**建议改进：**
```java
@Override
public void deleteProducts(Iterable<Long> productIdList) {
    Long currentUserId = UserContext.getUserId();
    String currentUserRole = UserContext.getRole();
    
    for (Long productId : productIdList) {
        Product product = productRepository.findById(productId)
            .orElseThrow(() -> new BizException(ResultCodes.NOTFOUND));
        
        // 权限检查
        if (!product.getSellerId().equals(currentUserId) && !"ADMIN".equals(currentUserRole)) {
            throw new BizException(ResultCodes.FORBIDDEN, 
                "无权删除商品: " + product.getName());
        }
    }
    
    productRepository.deleteAllById(productIdList);
}
```

### 3. Controller层的权限注解

**当前状态：**
- Service层有权限检查
- Controller层可以添加`@RequireRole`注解增强安全性

**示例：**
```java
@RequireRole(UserRole.SELLER)
@PostMapping("/product")
public Result<Product> addProduct(@RequestBody ProductDTO product) {
    // 只有SELLER和ADMIN可以访问此接口
    return Result.success(productService.addOneProduct(product));
}
```
